# hidashimora.github.io

## VPN подписка — без лагов на слабых устройствах

Исходные списки подписок часто содержат **больше 1000 серверов**. Телефоны и другие устройства с этим не справляются: приложение тормозит, долго грузит список, вылетает или просто не успевает обработать столько конфигов. Поэтому здесь всё настроено так, чтобы **уменьшить нагрузку** и убрать лаги.

- Берётся только **примерно четверть** конфигов из [исходного списка](https://raw.githubusercontent.com/AvenCores/goida-vpn-configs/refs/heads/main/githubmirror/26.txt) — устройству не нужно держать в памяти сотни лишних серверов.
- Список обновляется **автоматически** (при пуше и раз в час через GitHub Actions), без ручного вмешательства.
- Ничего вручную добавлять не нужно: достаточно **запушить репозиторий на GitHub** — workflow сам заполнит `vpn.txt`.

**Ссылка для VPN (подписка):**  
https://hidashimora.github.io/vpn.txt  

Raw: `https://raw.githubusercontent.com/hidashimora/hidashimora.github.io/main/vpn.txt`

---

**Как это работает:** при пуше в `main` и по расписанию (раз в час) workflow скачивает исходный список, оставляет только строки с конфигами (vless, trojan, vmess, ss, hy2), берёт каждую 4‑ю строку и сохраняет в `vpn.txt`. PHP и свой сервер не нужны — всё делает GitHub Actions.
